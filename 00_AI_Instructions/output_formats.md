# 出力フォーマットガイド（Excel / HTML / Markdown / PDF / PPTX）

> **正本は `02_Estimation_Template/estimate_master.md`（明細の構造化データ）。** 他形式はそこから派生生成する（Rule 4）。
> 形式を直接編集して内容を変えない。**正本を直して各形式を作り直す**のが原則（内容の一元管理）。
> どの形式でも 7 レイヤー（Rule 4）を保ち、**社内用と提出用を分ける**（Rule 10）。

## 0. 形式の選び方（用途別）

| 用途 | 第一候補 | 理由 |
|---|---|---|
| 工数集計・社内検討・機能 ON/OFF の再計算 | **Excel** | 計算式で連動、既存の運用と親和、単価変更が即反映 |
| 編集の正本・履歴管理 | **Markdown** | 誰でも編集可・差分が見やすい（`estimate_master.md`） |
| 顧客提出（見やすい・図つき）・軽く PDF 化 | **HTML** | ブラウザで開くだけ・ブランド統一・印刷で PDF |
| 顧客提出・清書（体裁固定・ページ番号） | **PDF** | 体裁が固定され提出に向く |
| 説明会・提案プレゼン | **PPTX** | 1 論点 1 スライドで口頭説明向き |

複数同時に出してよい（例: 社内検討 Excel + 提出 PDF + 説明 PPTX）。

> **内部/提出の分離（必須・Rule 10）**: どの形式でも「社内用（原価・利益率・社内単価・個人名あり）」と「提出用（それらなし）」を**別ファイル**で出す。提出用に社内情報を残さない。

## 0.5 ブランド統一（全形式共通）— 会社の標準で出す

**どの形式で出しても、必ず bravesoft 公式ブランドで統一する。** ブランドの正本は次の 2 つ:

- **`DESIGN.md`（ルート）** … 仕様の正本（色・タイポ・コンポーネント・do/don't）
- **`assets/brand.css`** … 実装の正本（CSS 変数 + コンポーネント。HTML 出力の共通土台）

### ブランド早見表（全形式でこの値を使う）

| 役割 | 値 |
|---|---|
| キーカラー（primary） | `#0066FF` |
| ボタン（resting / hover） | `#2966FF` / `#2247FF` |
| 補助ブルー（deep / sky / link） | `#3366FF` / `#3F9CFF` / `#0665FF` |
| トーナル背景（soft） | `#E5F0FF` |
| スタッツ・濃色面（navy） | `#253A58` |
| 本文インク / 濃インク | `#090A0A` / `#161717` |
| グレーランプ（muted / hairline 等） | `#6C7072` / `#72777A` / `#E3E5E5` |
| サーフェス（白 / 淡 / chip） | `#FFFFFF` / `#F7F9FA` / `#F2F4F5` |

> **広い色は使わない**（緑・橙・紫などの装飾色を足さない）。強調はブランド青＋ニュートラルで作り、status を示すときだけ success/error を**ラベル併記で**使う。

### 形式ごとの「ブランドの効かせ方」

| 形式 | ブランド適用の方法 |
|---|---|
| **Excel(.xlsx)** | **bravesoft 標準テンプレート**（`excel_format_spec.md` / `05_Tools/generate_estimate_excel.py`）。ヘッダ行 `#0066FF`(白文字)・紺帯 `#253A58`・入力欄 `#E5F0FF`・罫線 `#E3E5E5`。金額は桁区切り。社内用と提出用でシート/ファイルを分ける |
| **HTML** | `estimate_submission.html` / `estimate_internal.html`（`:root` が `brand.css` と同一トークン）を複製して流し込む。配布用は自己完結（インライン） |
| **PDF** | 上記 HTML を「印刷 → PDF」（最も確実）。`pdf` スキルで清書する場合も上表のトークン・フォント・ロゴを適用 |
| **PPTX** | 提案書テンプレ（ブランド済みマスター）を複製して文言差し替え（`docx`/`pptx` スキル） |
| **Word(.docx)** | `docx` スキル。見出し色 `#0066FF`・本文 `#090A0A`・表ヘッダ `#161717`・フォント Noto Sans JP・先頭ロゴ |
| **Markdown** | 正本。色は素では持たないが、各形式変換時に上記で付く |

→ これで **Excel / HTML / Markdown / PDF / PPTX / Word すべてが同じ bravesoft カラーで統一**して出せる。

## 1. Excel（計算・社内検討の主力）— bravesoft 標準テンプレート

- **仕様の正本は [`02_Estimation_Template/excel_format_spec.md`](../02_Estimation_Template/excel_format_spec.md)、生成は `05_Tools/generate_estimate_excel.py`（正本 md → JSON → .xlsx）。** 誰が作っても同じシート構成・同じ数式で出る。見本は `02_Estimation_Template/estimate_template.xlsx`。
- シート構成: **使い方 / 設定 / 前提条件 /（案比較）/ 見積サマリ[_案] / 見積書[_案] / スケジュール[_案] / ヒアリングシート / 非機能要件チェック**（社内の実案件 Excel を標準化）。
  - **見積サマリ**（見積書の隣・数式連携）＝案件概要＋金額サマリ（工数/税抜/税込/採算）。開いた瞬間に合計が見える。**見積書**＝顧客に渡す見積明細（社外金額を主役に、社内金額・利益率・根拠・MWO は折りたたみ）。
  - **ヒアリングシート**＝お客様ヒアリングの観点漏れ防止チェック（51+3 項目）。見積もりから分かる項目は AI が回答を事前記入し、人は「確認」列に ✓ を付けるだけ（✓ で行が色付く）。
- **設定シート＝変数**: 役割別単価（社内/社外）・管理費率・バッファ係数・税率・値引き・体制維持費を青セルに集約。ここを変えると全シートが連動（動的）。
- **計上チェック列（✓/－）**: 明細の行を金額から外すと、見積サマリ・PM 費・税込合計・スケジュールまで自動再計算 → スコープ調整（Step 05）が 1 クリック。行は消さずグレー表示（★スコープ未確定の 0 計上にも使う）。
- **社内/提出の分離は「列の折りたたみ」で実現**: 見積書は社外金額（税抜）を主役の I 列に、社内金額・利益率・根拠・MWO・採算ブロックを折りたたみグループに置く。このワークブックは**社内用**（社内単価・利益率入り）。顧客提出は下記 HTML/PDF が標準、または見積書の社内列を折りたたんで印刷範囲 A:I を PDF（Rule 10）。

## 2. Markdown（正本）

- `estimate_master.md` がそのまま正本。明細はテーブルで管理し、ここを直して各形式を再生成する。
- 👤 マークの行（原価・利益率・個人名）は提出用派生時に**必ず除外**する。

## 3. HTML / PDF（提出の主力）

- `estimate_submission.html`（提出用）/ `estimate_internal.html`（社内用）に流し込む。ブラウザ表示・印刷 PDF。
- スケジュール（月次ガント）は `schedule_submission.html`（bravesoft ブランド済み・営業構造）に流し込む。体制図は HTML 表 / Mermaid / SVG で。すべてブランド配色で。

## 4. PPTX（提案書）— 標準ジェネレータで生成

- **提案書 PPTX は `05_Tools/generate_proposal_pptx.py` で生成**（Excel と同じ見積もり JSON を入力＝1 つの正本から派生）。見本は `02_Estimation_Template/proposal_template.pptx`。
  ```bash
  python3 05_Tools/generate_proposal_pptx.py --input <見積もり.json> --out docs/estimates/<案件>_提案書.pptx
  ```
- **明るく・図解の多い・一目で分かる**トーン（bravesoft ブランド）。スライド構成（データがあるものだけ出す）:
  表紙 → 本ご提案のポイント（総工数/期間/主要機能/提示額の4スタット）→ セクション区切り 01〜04 →
  安心の3つの理由 → 費用構成ドーナツ → スコープ(Must/Want) → 進め方(単一=フェーズ流れ / 複数案=案比較カード) →
  お見積もり内訳（表＋合計カード）→ 工数内訳（横棒）→ 概算スケジュール（ガント）→ まとめ → クロージング。
- **この提案書は「提出用」**＝社内原価・利益率は載せない（Rule 10）。表紙の宛先・案件名は入力 JSON の `meta` に従う（テンプレは `○○` プレースホルダ）。
- 金額の詳細明細は Excel「見積書」/PDF を別添にし、スライドはサマリに留めると見やすい。
- 依存: `python-pptx`（必須）、`Pillow`（濃色スライドの白ロゴ用・無ければロゴ省略で継続）。Word が必要な場合は `docx` スキルで別途作成。

## 5. ファイル命名・保存

- 保存先: `docs/estimates/`
- 命名: `<案件名>_<社内 or 提出>_<YYYYMMDD>_v<版>.<ext>`（例: `会員アプリ_提出_20260625_v1.pdf`）
- 機密（社内用・原価入り）は `.gitignore` 対象。共有するものだけ `git add -f`。
