## 1. 評価（5 観点）

各観点に **🟢 良好 / 🟡 要注意 / 🔴 危険** と理由を付ける。

| 観点 | 判定 | 理由・根拠 |
|---|:--:|---|
| **A. 実現可能性（技術）** | 🟢🟡🔴 | 未知/高難度要素（リアルタイム・決済・大規模データ・独自SDK 等）の有無 |
| **B. 実現可能性（体制・期間）** | 🟢🟡🔴 | 必要体制（アプリ/サーバ/デザイン/QC）を期間内に組めるか、締切の現実性 |
| **C. リスク** | 🟢🟡🔴 | 要件の曖昧さ / 金銭・決済系 / 外部依存・他社連携 / スコープ膨張 / 既存改修の負債 |
| **D. 採算の見立て（粗い）** | 🟢🟡🔴 | 規模感（人日）と予算感の整合。`pricing_and_rates.md` で粗く換算 |
| **E. 前提充足度（インプットの質）** | 🟢🟡🔴 | 見積もりに足る情報が揃っているか、不足は何か |
