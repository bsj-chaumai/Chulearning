# 提出前チェックリスト

> Step 09 で**外部提出の直前に必ず通す**（Rule 13）。BrSE 旧テンプレの「見積提出前チェックリスト」を踏襲。
> **このチェックの後、人間レビューを必ず挟む。AI が独断で提出しない。**

---

## A. メタ・表紙（レイヤー①）

- [ ] 見積タイトル（何の見積もりか）が明記されている
- [ ] 宛先（会社名）が正しい
- [ ] 案件名 / システム名が正しい
- [ ] 作成日・**見積有効期限**・**見積精度（±%）**が入っている
- [ ] 通貨・税区分（税抜/税込）が明示されている
- [ ] バージョン（版）が更新されている

## B. 前提・スコープ（レイヤー②③）

- [ ] 見積根拠（参照資料）が書かれている
- [ ] 対応範囲（工程の ON/OFF）が明示されている
- [ ] **対象外（Out of Scope）**が明示されている
- [ ] 契約形態（準委任/請負）が書かれている
- [ ] 対応 OS・端末・環境が書かれている
- [ ] 納品物・検収条件・ソースコードの扱いが書かれている

## C. 明細・金額（レイヤー④⑤）

- [ ] 各項目に工数(人日)・単価・小計がある（根拠が説明できる＝ Rule 1）
- [ ] PM・管理費 / テスト / リリース・ストア申請 / 非機能 が入っている（Rule 2）
- [ ] 集計（小計→管理費→バッファ→値引き→税→合計）が正しい
- [ ] 数字の整合（明細合計＝集計、Phase 別合計＝全体）が取れている

## D. 機密（最重要・Rule 10）

- [ ] **提出用に原価・社内単価が入っていない**
- [ ] **提出用に利益率が入っていない**
- [ ] **提出用に担当者の個人名・他案件の情報が入っていない**
- [ ] 社外秘資料の中身が成果物に混入していない

## E. 体裁・ブランド

- [ ] bravesoft 公式デザイン（#0066FF / Noto Sans JP / 共通ロゴ）で統一されている
- [ ] 誤字・桁・単位（円/人日）の確認をした
- [ ] スケジュール・体制（必要な場合）が添付されている

## F. 最終

- [ ] 🔔 **人間レビューを得た**（外部提出の最終承認）
- [ ] 保存先（`docs/estimates/`）と機密の除外設定（`.gitignore`）を確認した

---

> 1 つでも未チェックがあれば提出しない。特に **D（機密）** は提出事故に直結するため必ず全項目確認する。
