5. Đề xuất cải tiến & tính năng mới (ưu tiên)

  1. Gộp luồng nhập liệu (Must) — đúng hướng mockup: hết trùng tab, preview lãi live.
  2. Cài đặt chung tỷ giá / phí / hệ số sỉ (Must) — giảm lỗi nhập tay.
  3. Thống nhất công thức lãi (Must) — một engine dùng cho list + breakdown.
  4. Lưu cloud + tìm kiếm/export (Must) — dùng được nhiều máy.
  5. Mô phỏng đổi giá bán (Want) — quyết định giá nhanh hơn.
  6. Bảng giá sỉ xuất PDF (Want) — gửi khách sỉ.
  7. UI song ngữ VI/JA (Want) — khớp cách làm việc JP–VN.
  8. Shopee API / tồn kho (Out → Phase sau) — khi vận hành ổn định.

2. 入力フロー最適化モック

3タブを1ウィザード／1商品マスタに統合。共通設定の為替・手数料プロファイル、利益のリアルタイムプレビュー。

モックを全画面で開く AS-ISプロトタイプと比較

為替・手数料を毎回入力共通設定＋手数料プロファイル選択

5. 改善・新機能提案（優先順）

  1. 入力フロー統合（Must） — モック方針：タブ重複解消、利益ライブ表示。
  2. 為替／手数料／卸係数の共通設定（Must） — 手入力ミス削減。
  3. 利益計算式の一本化（Must） — 一覧と内訳で同一エンジン。
  4. クラウド保存＋検索／エクスポート（Must） — 複数端末利用。
  5. 売価シミュレーション（Want） — 価格決定を高速化。
  6. 卸価格表PDF（Want） — 卸先への共有。
  7. UIのVI/JA切替（Want） — 日越協業に適合。
  8. Shopee API／在庫（Out→後続） — 運用安定後。

7. 非機能（要約）

  • 性能: プレビュー再計算 < 50ms（約500商品）
  • セキュリティ: 認証/クラウド時はHTTPS、料率・仕入情報の露出防止
  • 可用性: 定期バックアップ／エクスポート
  • 拡張性: 手数料プロファイルの版管理、将来のAPI接続余地
