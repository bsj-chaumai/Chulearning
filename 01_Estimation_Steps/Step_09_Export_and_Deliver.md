# Step 09: 出力 & 納品

## 1. 目的

確定した見積もり・提案を、利用者が選んだ形式（**Excel / HTML / Markdown / PDF / PPTX**）で出力し、**提出前チェックリスト**と**人によるレビュー**を通してから納品する（Rule 6 / Rule 13）。

## 2. 前提条件

- Step 08 完了（社内用/提出用・提案書が揃っている）。
- 参照: `00_AI_Instructions/output_formats.md`、`03_Assessment_and_Checklists/pre_submission_checklist.md`。

## 3. ヒアリング項目

- 出力形式（複数可）と、それぞれの用途（社内検討 / 顧客提出 / 説明会）
- 提出先・提出方法（メール添付 / 共有リンク 等）

## 4. 作業内容

1. **形式出力**: 正本（estimate_master）から各形式を派生（`output_formats.md` 参照）。
   - **Excel**: **標準テンプレートで生成**（`02_Estimation_Template/excel_format_spec.md` ＝仕様 / `05_Tools/generate_estimate_excel.py` ＝生成）。**「見積サマリ」**（案件概要＋金額サマリ・見積書の隣・数式連携）と**「見積書」**（顧客に渡す明細・社外金額を主役に社内金額/利益率は折りたたみ）に分離。ほかに 前提条件・スケジュール・**ヒアリングシート（観点チェック）**・非機能要件チェックを含む。設定シート（単価・管理費率・バッファ・税率）と計上チェック列（✓/－で行を金額から除外→全連動）を備える。社内用（Rule 10。顧客提出は社内列を折りたたんで印刷範囲 A:I/A:D を PDF、または下記 HTML/PDF）。
   - **HTML/PDF**: 提出用レイアウト（提出用見積・見積根拠資料）。HTML を印刷 → PDF。
   - **PPTX**: 提案書（説明会用）。
   - **Markdown**: 編集の正本。
   - すべて bravesoft ブランド（`DESIGN.md` / `brand.css`）で統一。
2. **提出前チェックリスト**: `pre_submission_checklist.md` を通す（宛先/案件名、前提・対象外・契約形態、納品物、有効期限・見積精度、**機密混入なし**、数字整合 等）。
3. **🔔 人間レビュー（必須・Rule 13）**: 外部提出前に人間の承認を得る。**AI が独断で提出・確定しない。**
4. 成果物を `docs/estimates/<案件>_<YYYYMMDD>.<ext>` に保存。共有するものだけ `git add -f`（Rule 10 / `.gitignore`）。

## 5. 完了条件チェックリスト

- [ ] 指定形式（Excel/HTML/PDF/PPTX/MD）で出力した
- [ ] すべて bravesoft ブランドで統一されている
- [ ] 提出前チェックリストを全項目通した
- [ ] 提出用に原価・利益率・社内単価・個人名が混入していない（Rule 10・最終確認）
- [ ] 🔔 外部提出前の人間レビューを得た（Rule 13）
- [ ] `docs/estimates/` に保存し、機密は除外設定を確認した

## 6. `ai_context_log.md` 更新事項

- §5 Step 完了履歴（出力形式・納品先・最終金額）
- §10 次への申し送り（改版が来た場合の対応方針）

## 7. 次 Step への引き継ぎ事項

**AI → AI 引き継ぎ**: 改版（値引き交渉・スコープ変更）が来たら、正本（estimate_master）を直して各形式を再生成する（形式側を直接いじらない）。版管理（作成日・バージョン）を更新する。

**AI → 人間 次アクション**（Section 11.7 / 11.8 適用例）:

```
📤 見積もり / 提案を出力しました
- 形式: <Excel / HTML / PDF / PPTX>（社内用 / 提出用を明示）
- ファイル: docs/estimates/<案件>_<日付>.<ext>
- 提出前チェック: 全項目 PASS / 一部 要対応(<列挙>)
- 機密チェック: 提出用に原価・利益率・個人名の混入なし ✅

🔔 外部提出前レビュー: 提出してよい内容かご確認ください（最終承認はあなた）
⏭ 次にあなたにやってほしいこと:
1. 提出物を開いて最終確認
2. 別形式が必要なら指定 / 改版があれば指示

🤖 回答後に AI がやること: 指摘修正 / 追加形式の生成 / 改版対応
```
