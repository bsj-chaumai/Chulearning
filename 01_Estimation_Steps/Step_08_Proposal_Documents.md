# Step 08: 提案ドキュメント生成

## 1. 目的

確定した見積もり（明細・金額・スケジュール）から、**顧客に渡せる提案ドキュメント**を生成する。**内部用（原価・利益率つき）と提出用（それらを出さない）を分離**する（Rule 4 / Rule 10）。

主な成果物:
- **提出用見積もり**（項目・概要・工数・金額＋前提・スコープ・スケジュール）
- **見積根拠資料**（各項目の工数根拠を、検索・絞り込みできる形で）
- **提案書**（PPTX / Word：背景・目的・提案概要・スコープ・体制・スケジュール・見積もりサマリ）

## 2. 前提条件

- Step 05〜07 完了（金額確定・クロスチェック済み）。
- 参照: `02_Estimation_Template/`（estimate_internal / estimate_submission）、`00_AI_Instructions/output_formats.md`、`assets/`（ロゴ・brand.css）。

## 3. ヒアリング項目

- どの成果物が要るか（提出用見積もり / 見積根拠資料 / 提案書）
- 提出先・トーン（社内検討 / 顧客提出 / 経営説明）
- 複数案（Phase 別・スコープ別）を併記するか

## 4. 作業内容

1. **内部/提出の分離**: 正本（estimate_master）から 2 系統を派生。
   - **社内用** (`estimate_internal`): 原価・利益率・社内単価・担当名を含む（社内限り）。
   - **提出用** (`estimate_submission`): 項目・概要・工数・金額・前提・スコープ・スケジュールのみ。**原価/利益率/社内単価/個人名/他案件を出さない**（Rule 10）。
2. **見積根拠資料**: 各項目の「なぜこの工数・金額か」を一覧化し、Step 07 の想定問答を添える。検索・絞り込み・印刷できる HTML が望ましい（「○○高くない？」への即答用）。
3. **提案書（PPTX）**: `05_Tools/generate_proposal_pptx.py` で生成（Excel と同じ見積もり JSON が入力＝1 つの正本）。見本 `02_Estimation_Template/proposal_template.pptx`。表紙 → 本ご提案のポイント → 安心の根拠 → 費用構成ドーナツ → スコープ(Must/Want) → 進め方/案比較 → お見積もり内訳 → 工数内訳 → 概算スケジュール（ガント）→ まとめ → クロージングの、**明るく図解の多い**構成。**提出用**なので社内原価・利益率は載せない（Rule 10）。`python3 05_Tools/generate_proposal_pptx.py --input <json> --out docs/estimates/<案件>_提案書.pptx`。Word が要る場合は `docx` スキルで別途。
4. すべて bravesoft 公式デザイン（`#0066FF` / Noto Sans JP ＋ Roboto Mono / 共通ロゴ）で出す。

> 提案書は「人が最終ブラッシュアップするだけで提出できる状態」を目指す（ノンコア工数の削減）。
