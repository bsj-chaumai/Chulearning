# Step 06: スケジュール & 体制

## 1. 目的

確定した見積明細（工数）から、**WBS / ガント・体制（役割別アサイン）・Phase/STEP 分割・総期間**を**派生生成**する。
見積もりとスケジュールを別作業で二度作る手間をなくす（現場の不満を解消）。

## 2. 前提条件

- Step 05 完了（明細・合計・Phase/スコープ案が確定）。
- 参照: `docs/estimation_brief.md`（締切・リリース希望）、見積明細（フェーズ別工数）。

## 3. ヒアリング項目

- プロジェクト開始日（または希望リリース日から逆算）
- 並走人数（役割別の同時稼働人数：PM / デザイナー / SE / アプリ / QC 等）
- フェーズの開始ラグ（前フェーズ完了を待たず並走を始める割合）
- 受入テスト・ストア審査・脆弱性診断など顧客側/待ち時間の見込み

## 4. 作業内容

1. **WBS 化**: 見積明細のフェーズ別工数を工程行に集約（要件定義 / デザイン / 設計 / 開発 / テスト / 受入 / 脆弱性診断 / ストア申請・審査待ち / リリース）。
2. **期間計算**: 各工程の `工数(人日) ÷ 並走人数 = 営業日`、開始ラグを足して開始日/終了日を算出。営業日/月（標準 20）で月数換算。
   - **開始日・並走人数・工数を変えると全体が連動**する「動的スケジュール」を志向する（HTML/Excel で入力欄→自動計算）。
3. **体制（役割別アサイン）**: 役割ごとの人数・稼働を提示。社内の原価管理（人月×稼働率）は社内用にとどめる（Rule 10）。
4. **Phase / STEP 分割**: Step 05 の複数案（Phase1=MVP / Phase2=拡張 等）に対応したスケジュールを出す。
5. **総期間・マイルストーン**: 着手〜リリースの総営業日・カ月、主要マイルストーン（要件 Fix / デザイン Fix / 開発完了 / 受入 / リリース）を明示。
6. ガントは図で表現（Mermaid `gantt` / HTML 表 / SVG）。**提出用は `02_Estimation_Template/schedule_submission.html`**（bravesoft ブランドの月次ガント：No./工程/担当/Start/Due/日数 ＋ 月×週バー、工程グループ行・◆マイルストーン・工程別色・保守=別途グレー）に流し込む。見積もり（`estimate_submission.html`）と対の提出セット。
