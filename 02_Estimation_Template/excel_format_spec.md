# 標準見積もり Excel 仕様 — excel_format_spec

> **キットの Excel 出力（社内用・計算の主力）の統一仕様。** 社内で最も作り込まれた実案件見積もりの Excel 構成を全社標準化し、「計上チェック」と「設定シート」を加えたもの。
> 生成は `05_Tools/generate_estimate_excel.py`（正本 `estimate_master.md` → JSON → .xlsx）。手で作る場合も本仕様に合わせる。
> **提出物の標準は従来どおり `estimate_submission.html`（HTML → PDF）**。この Excel は社内検討・スコープ調整・採算確認の主力（Rule 10: 社内金額・利益率入りのため社内用）。

見積書の右側（社内グループ内）には **💡 MWO 別金額**・**🔍 抜け漏れチェック（PM・テスト・リリース/ストア申請・非機能の計上を自動判定）**・**フェーズ×カテゴリ 工数マトリクス**（空欄＝工数ゼロ＝抜け候補）を内蔵。カテゴリ列は Rule 2 の標準分野（PM・全体管理/要件定義/UI/UXデザイン/外部設計/内部設計/アプリ/フロントエンド/API・サーバ/管理画面/インフラ/バッチ/外部連携/テスト/セキュリティ/運用・保守/共通）のプルダウン＋自由入力。

複数案（Phase1/Phase2/一括 等）は `見積書_Phase1` のようにシートを並べる（実案件の 3 パターン併記を踏襲）。設定・前提条件シートは全案共通。

## 2. 見積書（見積明細テーブル）の列仕様

> **顧客に見せる列（A〜I）を左に、社内列（J〜L）＋右側ブロックを折りたたみグループに。** 顧客提出は印刷範囲 A:I を PDF（社内金額・利益率は出ない＝Rule 10）。金額サマリ・案件概要は隣の「見積サマリ」シート（数式連携・下スクロール不要）。

| 列 | 内容 | 備考 |
|---|---|---|
| A | **計上**（✓ / －） | プルダウン。**－にすると行の金額が 0 になり、見積サマリ・PM費・税込合計・スケジュールまで全連動**。行は削除せずグレー表示。★スコープ未確定は － のまま一覧に残す |
| B/C | フェーズ / カテゴリ | Rule 2 のマトリクス軸 |
| D | 項目 | 大機能は **親行「…【内訳合計】」＋子行「　└ 」**（工数・金額は子行から自動合算、集計で二重計上しない） |
| E | 概要 | 顧客にも見える説明 |
| F | 実装方式 | ネイティブ / WebView / 外部ブラウザ / 共通 / －（ハイブリッド案件の肝） |
| G | 工数(人日) | 青セル＝入力。**PM 行だけは「率」**（下記） |
| H | 担当（役割） | プルダウン（設定シートの単価表と連動） |
| **I** | **金額(税抜)＝社外** | `=工数 × VLOOKUP(担当, 単価表, 社外単価)`（計上－なら 0）。**顧客に見せる主役の金額列** |
| J | 社内金額 👤 | 同・社内単価。**折りたたみグループ内（社内限り・印刷範囲外）** |
| K | 根拠・備考 | **Rule 1: なぜこの工数か**を必ず書く（グループ内） |
| L | **優先度（MWO）** | **金額を動かすレバー**。M=必須／W=できれば（＝予算調整の余地・青太字）／O=対象外（グレー）／★未確定=0 計上（赤太字）。右側の **💡 MWO 別金額ブロック**（社内グループ内）に「M だけの金額」「W の合計（＝削れる額）」「O・★への金額混入チェック」「Must 構成の提示額（概算）」を自動集計。予算調整は ① 💡 で W 合計を見る → ② W の行の計上を － にする（値引きより先）（グループ内） |

- **見積サマリ（別シート・連携）**: 案件概要＋「工数(計上分)・金額(税抜=社外)・社内金額👤・利益率👤」（フェーズ別＋全体）／金額構成（明細小計 → バッファ → 体制維持費 → 値引き → **ご提示額 税抜/消費税/税込**）／採算 👤（利益率: 工数どおり・バッファ消化時の 2 本＝Rule 5）。見積書の明細を全列 SUMIFS で参照するため、明細を直すと自動追随。**開いた瞬間に合計が見える**（顧客提出は A:D、採算は範囲外）。
- **PM・全体管理行**: 各フェーズ先頭に 1 行。G 列に率（%表示・既定は `=管理費率`＝設定連動、行ごとに上書き可）。金額は「フェーズ内 leaf 項目の合計 ×率」を自動計算（親行は控除済み）。`CELL("format")` ハックは廃止し明示的な数式に置換。
- **社内列の折りたたみ**: J〜L と右側ブロック（N 列〜）を Excel の列グループ（アウトライン）にまとめ、列上の [－] でワンクリック非表示。顧客提出用の印刷範囲は A:I（横 1 ページに fit・landscape）で、社内金額・利益率は出力されない。
- **隠し補助列 BH/BI/BJ（行種別 leaf/pm/parent と leaf 金額）**: 全集計は **行種別列の完全一致×全列 SUMIFS**（leaf＋pm を加算、parent 除外、サマリ行は BH 空で自動除外）→ 行の追加・挿入に追随し、項目名に依存せず循環参照もしない。
- **フェイルセーフ**: 担当が空欄/単価表に無い場合は金額 0＋担当セルを赤表示（#N/A を合計に伝播させない）。数式セルはシート保護（パスワード無し・行挿入や書式変更は許可）。

## 3. 変数（設定シート）— ここだけ変えれば全部連動

| 変数（定義名） | 既定 | 効く場所 |
|---|---|---|
| 単価表（役割別 社内/社外・円/人日） | pricing_and_rates.md の標準 | 明細 I/J・PM 費・全集計 |
| 管理費率 | 20%（20〜30%） | 全 PM 行（行単位で上書き可） |
| バッファ係数 / バッファ適用（✓/－） | 1.2 / － | 金額サマリのリスクバッファ行 |
| 体制維持費率（任意） | 0%（例 5%） | 金額サマリの体制維持費行 |
| 税率 | 10% | 消費税・税込 |
| 値引き | 0 | 金額サマリ（第一候補はスコープ調整＝計上チェック） |
| 営業日/月 | 20 | 人月換算・スケジュール |

数式は日本語の**定義名**（`管理費率`・`単価表` 等）で設定シートを参照する。

## 3.5 標準明細カタログ（テンプレは「全部入りから削る」）

`estimate_template.xlsx` / `05_Tools/sample_input.json` には、**よくある見積もり項目 約35行をプリセット済み**（要件定義5・デザイン4・設計5・開発11（親子行/管理画面/プッシュ/バッチ/★外部連携含む）・テスト5・リリース2＋各フェーズPM行）。ゼロから書かず、**「消す・計上－にする・工数と※印の項目名を書き換える」だけ**で見積もりが立ち上がる（全部入りから削る方が抜け漏れが出ない＝Rule 2）。役割は PM/Dir/SE/デザイナー/アプリ/フロントエンド/バックエンド/オフショア/QC の 9 種を標準装備。

## 4. 操作規約（誰でも同じ品質で使うために）

1. **青セルだけ触る**（入力欄）。白の数式セルは保護済み（文字欄＝項目・概要・担当・根拠などは白でも編集可）。
2. スコープ調整・値引き交渉は、**行を消さず「計上 －」**で外す（Rule 5 / Step 05）。
3. 行追加は**既存の明細行の間に挿入して上の行をコピー**（数式・隠し列ごと）。大量追加・フェーズ追加は AI に再生成を依頼するのが確実（見積サマリ・PM費・税込は自動追随する）。
4. 単価・率を標準から変えたら理由を `ai_context_log.md` §6 に残す（Rule 9）。
5. **このファイルは社内用**（社内単価・利益率入り）。顧客提出はキットの提出用 HTML/PDF が標準。Excel で渡す必要がある場合は「見積書」の社内列（J〜L・右側ブロック）を折りたたみ、印刷範囲 A:I を PDF 出力（Rule 10・提出前チェックリスト必須。見積書上部にも警告文あり）。

## 5. ブランド

ヘッダ行 `#0066FF`（白文字）／紺色 `#253A58`（見出し・強調）／入力欄 `#E5F0FF`／小計帯 `#F2F4F5`／罫線 `#E3E5E5`／フォント Noto Sans JP（未導入環境はゴシックに自動代替）。金額は `¥#,##0`。

## 6. 検証済み事項（2026-07-05・多角検証4観点＋回帰テスト）

- 実案件 Phase1（67 項目・親子行・PM30%・役割別単価・スケジュール 13 行・機能×API 70 行）を本標準で再生成し、**元 Excel と全フェーズ小計・工数・合計（税抜 40,787,500 / 税込 44,866,250）が完全一致**。
- 計上チェック OFF → 明細 → PM 費 → 見積サマリ（別シート・連携）→ 税込合計まで連動を確認。
- シナリオテスト全通過: バッファ適用（×1.2）・値引き・単価変更・管理費率変更・開始日/並走人数変更が全シートに正しく連動。
- エッジケース: 担当不正＝¥0＋赤表示（#N/A 伝播なし）／計上 OFF＝グレー＋取り消し線・金額 0／最下行より下への追記も見積サマリ・集計に反映。
- LibreOffice 再計算で**数式エラー 0**（XLOOKUP 不使用・VLOOKUP/SUMIFS/WORKDAY 等の互換関数のみ。ワイルドカード条件も不使用＝Calc の計算オプション差に非依存）。

### 既知の制約
- 見積書は明細＝顧客提出物そのものなので、行の増減が即サマリ・合計に反映（別シート同期は不要）。
- 親行【内訳合計】の子行範囲は連続行前提。子行の追加は子行の間に挿入する。
- ガントの月列は 18 ヶ月分（超える場合は右へ列コピー）。`TEXT` の書式コードは日本語/英語 UI 前提（国内利用では問題なし）。
